# Superstore Sales Dashboard – Tableau Project

## Overview
This project analyzes a Superstore Sales dataset using Tableau to uncover business insights related to sales performance, regional trends, product categories, and forecasting. The dashboard is designed to help stakeholders monitor key metrics and make data-driven decisions.

## Project Objectives
- Analyze overall sales performance
- Identify top-performing product categories
- Compare sales across regions
- Visualize geographic sales distribution
- Forecast future sales trends
- Build an interactive dashboard for business reporting

---

# Dataset Information
The dataset used for this project contains transactional sales data from a retail superstore.

### Key Fields
- Order Date
- Sales
- Profit
- Quantity
- Region
- State
- Category
- Sub-Category
- Customer Segment
- Ship Mode

# Challenges Encountered

## 1. Data Cleaning Issues
Some fields contained missing or inconsistent values that affected visualization accuracy.

### Solution
- Cleaned the dataset before importing into Tableau
- Standardized field names and formats
- Removed duplicates and null values where necessary

---

## 2. Dashboard Performance
Large datasets caused slower dashboard loading times.

### Solution
- Reduced unnecessary calculations
- Optimized filters and visual elements
- Used extracts instead of live connections

---

## 3. Forecast Accuracy
Forecast results varied depending on the selected time range.

### Solution
- Adjusted date granularity
- Tested multiple forecasting periods
- Validated trends against historical data

---

## 4. Map Visualization Challenges
Some geographic locations were not automatically recognized by Tableau.

### Solution
- Corrected geographic roles
- Standardized state and region names
- Verified location data formatting
